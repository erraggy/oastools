# oastools Examples

This directory contains complete, working examples generated by oastools.

## Available Examples

### [petstore/](petstore/)

A full-featured client and server generated from the classic [Swagger Petstore API](https://petstore.swagger.io/).

**Source API:** https://petstore.swagger.io/v2/swagger.json (OAS 2.0)

**Generated with:**
```bash
oastools generate -server -server-all -client -security-enforce \
  -oauth2-flows -oidc-discovery -credential-mgmt \
  -p petstore -o ~/code/PetStore \
  https://petstore.swagger.io/v2/swagger.json
cd ~/code/PetStore
go mod init petstore
go mod tidy
```

**Features demonstrated:**
| Flag | Feature |
|------|---------|
| `-client` | HTTP client with typed methods for all operations |
| `-server` | Server interface with request/response types |
| `-server-all` | Router, middleware, binder, responses, and stubs |
| `-security-enforce` | Server-side security enforcement middleware |
| `-oauth2-flows` | OAuth2 token flow helpers |
| `-oidc-discovery` | OpenID Connect discovery client |
| `-credential-mgmt` | Credential provider interfaces |

**Generated files:**
- `types.go` - Schema model structs with validation tags
- `client.go` - HTTP client with methods for all API operations
- `server.go` - Server interface definition
- `server_router.go` - HTTP router using net/http
- `server_middleware.go` - Request/response validation middleware
- `server_binder.go` - Type-safe parameter binding
- `server_responses.go` - Typed response writers
- `server_stubs.go` - Stub implementations for testing
- `security_helpers.go` - Client authentication helpers
- `oauth2_petstore_auth.go` - OAuth2 client for petstore_auth
- `credentials.go` - Credential provider system
- `security_enforce.go` - Server security middleware
- `oidc_discovery.go` - OIDC discovery client
- `README.md` - Auto-generated usage documentation
